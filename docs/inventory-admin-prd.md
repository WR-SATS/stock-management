# KeyPal 库存管理后台产品需求文档

版本：v0.1  
日期：2026-04-30  
来源文件：`KeyPal 产品收发&库存&耗材 统计表-2026.xlsx`

## 1. 背景与目标

当前库存、出库、到货、耗材、售后、盘点等业务记录分散在 Excel 多个工作表中，部分库存和消耗依赖公式自动计算。随着 SKU、平台订单、仓库、耗材和售后记录增加，Excel 容易出现录入格式不统一、公式被误改、库存追溯困难、多人协作权限不清晰等问题。

本后台的目标是把 Excel 台账升级为可登录、可追踪、可统计的库存管理系统，支持管理者维护商品 SKU、登记商品出库和到货入库、自动计算库存和耗材消耗、汇总销售与库存风险，并管理售后异常。

## 2. Excel 现状分析

### 2.1 工作表结构

| 工作表 | 当前用途 | 后台模块映射 |
| --- | --- | --- |
| `SKU字典` | 商品/物料 SKU 基础信息，包含 SKU、名称、类型、颜色/版本、单位、出库列映射、启用、重量、尺寸 | SKU 管理、产品档案 |
| `出库表` | 销售/平台/展会等出库记录，包含发货日期、出库产品、订单金额、经手人、平台、平台编号、快递信息 | 出库管理、销售汇总、物流记录 |
| `KeyPal 2 到货记录` | 成品/物料采购到货、供应商、订购数量、本次到货、累计到货、质检 | 入库管理、采购到货、质检 |
| `KeyPal 2 耗材统计记录` | KeyPal 2 耗材计划、到货、自动消耗、当前库存、最近到货/消耗 | 耗材库存、BOM 消耗规则 |
| `库存` | 物料当前库存、安全库存、仓库、库位、低库存判断 | 库存总览、安全库存预警 |
| `异常以及售后` | 售后/漏发/设备异常等非结构化记录 | 售后管理、异常工单 |
| `盘点表` | 系统数量、实盘数量、差异、处理方式 | 库存盘点、库存调整 |
| `耗材到货记录` | 包装/天地盖等耗材初始数量、已用、剩余、备注 | 耗材到货与消耗 |
| `订单计划表` | 当前库存、安全库存、销量、预测需求、下单量 | 采购计划、补货建议 |
| `展会出库` | 展会、拆样等非订单型出库 | 出库场景分类 |

### 2.2 关键业务对象

| 对象 | 说明 |
| --- | --- |
| 管理者账号 | 可注册、登录，后续支持角色权限和操作日志 |
| SKU/产品 | 成品、耗材、物料等统一档案，包含名称、类型、颜色/版本、单位、重量、尺寸、启用状态 |
| 入库单 | 采购/生产/退回等入库记录，可关联供应商、质检、仓库库位 |
| 出库单 | 销售、展会、拆样、售后补发等出库记录，可关联平台订单、快递、经手人 |
| 库存余额 | 按 SKU、仓库、库位汇总当前库存，并与安全库存比较 |
| 耗材 | 可单独入库，也可根据成品到货或出库规则自动消耗 |
| 售后工单 | 记录问题、订单号、退回快递、处理状态、责任归类和解决结果 |
| 盘点单 | 定期录入实盘数量，产生差异并形成库存调整记录 |

## 3. 用户与权限

### 3.1 用户角色

| 角色 | 权限 |
| --- | --- |
| 超级管理员 | 管理账号、SKU、所有单据、库存调整、系统配置 |
| 仓库管理员 | 入库、出库、盘点、库存查看、耗材登记 |
| 销售/运营 | 创建销售出库、查看销售汇总、查看订单相关售后 |
| 售后人员 | 创建和处理售后工单，发起售后补发/退回入库 |
| 只读查看者 | 查看库存、销售、耗材和报表，无编辑权限 |

### 3.2 登录注册

MVP 支持：

- 管理者注册账号：手机号/邮箱、姓名、密码。
- 登录/退出登录。
- 密码加密存储。
- 首个注册账号默认为超级管理员，后续账号由超级管理员审核或分配角色。
- 记录关键操作日志：创建、修改、删除、审核、库存调整。

## 4. 功能需求

### 4.1 仪表盘

展示关键经营和库存指标：

- 今日/本月出库数量。
- 本月销售金额。
- 当前总库存 SKU 数。
- 低于安全库存的 SKU 数。
- 待处理售后数量。
- 近 30 天出库趋势。
- 低库存列表。
- 最近入库、最近出库、最近售后。

### 4.2 SKU 管理

字段建议：

| 字段 | 说明 |
| --- | --- |
| SKU | 唯一编码，如 `FG-KP2-BLK` |
| 名称 | 中文名称，如 `KeyPal 2 黑色-喷油` |
| 类型 | 成品、耗材、物料、配件 |
| 颜色/版本 | 黑色-喷油、红色-喷油、透明蓝色等 |
| 单位 | 台、个、套、张等 |
| 重量 | 可选 |
| 尺寸 | 可选 |
| 安全库存 | 库存预警阈值 |
| 启用状态 | 启用/停用 |
| 备注 | 其他说明 |

需求：

- 新增、编辑、停用 SKU。
- SKU 唯一性校验。
- 按类型、启用状态、关键词筛选。
- 支持从 Excel 的 `SKU字典` 初始导入。
- SKU 被单据使用后不可物理删除，只能停用。

### 4.3 商品出库管理

对应 Excel `出库表` 和 `展会出库`。

字段建议：

| 字段 | 说明 |
| --- | --- |
| 出库单号 | 系统生成 |
| 出库日期 | 原 `发货日期/真实日期` |
| 出库类型 | 销售、展会、拆样、售后补发、其他 |
| 平台 | 有赞、线下、展会等 |
| 平台编号 | 订单号 |
| 商品明细 | SKU、数量、单价、金额 |
| 订单金额 | 可自动汇总，可手工覆盖 |
| 经手人 | 操作/业务负责人 |
| 快递公司 | 顺丰等 |
| 快递单号 | 物流追踪 |
| 快递结算方式 | 到付/月结/现付等 |
| 快递费用 | 运费 |
| 备注 | 说明 |

需求：

- 创建出库单时选择 SKU 和数量，不再手输 `SKU*数量+SKU*数量` 格式。
- 保存后自动扣减库存。
- 库存不足时提示，可配置是否允许负库存。
- 支持一单多 SKU。
- 支持销售出库、展会出库、拆样、售后补发等类型。
- 支持按日期、平台、SKU、经手人、订单号、快递单号查询。
- 支持导出出库明细。

### 4.4 到货入库管理

对应 Excel `KeyPal 2 到货记录`。

字段建议：

| 字段 | 说明 |
| --- | --- |
| 入库单号 | 系统生成 |
| 到货日期 | 实际到货日期 |
| 供应商 | 供应商名称 |
| 采购品名 | 采购描述 |
| SKU/物料 | 关联 SKU |
| 工艺/材质 | 物料属性 |
| 订购数量 | 采购订单数量 |
| 本次到货数量 | 本次入库数量 |
| 累计已到货 | 同采购批次累计 |
| 未到货数量 | 订购数量 - 累计已到货 |
| 单价含税 | 采购单价 |
| 金额 | 数量 * 单价 |
| 外箱数 | 包装信息 |
| 抽检数量 | QC 抽检数 |
| 合格数量 | QC 合格数 |
| 不良数量 | QC 不良数 |
| 合格率 | 合格数量 / 抽检数量 |
| 质检结果 | 合格/让步接收/不合格 |
| 仓库/库位 | 入库位置 |
| 备注 | 说明 |

需求：

- 新增到货记录后增加库存。
- 支持采购批次累计到货和未到货计算。
- 支持质检信息录入。
- 不合格品可进入异常库存或生成售后/质量异常单。
- 支持按供应商、SKU、日期、质检结果查询。

### 4.5 库存管理

对应 Excel `库存` 和 `盘点表`。

需求：

- 按 SKU、仓库、库位展示当前库存。
- 当前库存由入库、出库、盘点调整等流水自动汇总。
- 显示安全库存和预警状态。
- 支持库存流水追溯：每个库存变化都能查看来源单据。
- 支持库存调整，需填写原因并记录操作人。
- 支持盘点单：
  - 创建盘点任务。
  - 导出/录入实盘数量。
  - 自动计算差异数量。
  - 填写差异原因、处理方式。
  - 审核后生成库存调整流水。

### 4.6 销售汇总

基于出库单统计：

- 按月统计销售金额、出库数量。
- 按 SKU 统计销量、金额。
- 按平台统计订单数量、金额、出库数量。
- 按经手人统计出库量。
- 支持日期范围筛选。
- 支持导出 Excel。

MVP 报表：

| 报表 | 维度 | 指标 |
| --- | --- | --- |
| 月度销售汇总 | 月份 | 出库数量、订单金额 |
| SKU 销售排行 | SKU | 出库数量、销售金额 |
| 平台销售汇总 | 平台 | 订单数、销售金额 |
| 出库明细 | 日期、订单、SKU | 数量、金额、物流 |

### 4.7 耗材管理

对应 Excel `KeyPal 2 耗材统计记录` 和 `耗材到货记录`。

字段建议：

| 字段 | 说明 |
| --- | --- |
| 耗材 SKU | 如 `KP2-SHELL-BLK` |
| 耗材名称 | 壳体-黑色、天地盖等 |
| 单位 | 套、个、张 |
| 计划数量 | 预计需求 |
| 累计到货 | 耗材入库累计 |
| 累计已用 | 按规则自动消耗或手动登记 |
| 当前库存 | 累计到货 - 累计已用 |
| 未到差额 | 计划数量 - 累计到货 |
| 最近到货日 | 自动取最近入库 |
| 最近消耗日 | 自动取最近消耗 |

需求：

- 维护耗材 SKU。
- 登记耗材到货。
- 登记耗材消耗。
- 支持 BOM 消耗规则：例如 KeyPal 2 黑色成品每到货或出库 1 台，消耗黑色壳体 1 套。
- 支持按产品颜色/版本配置耗材用量。
- 自动计算当前库存、未到差额、低库存提醒。

### 4.8 售后管理

对应 Excel `异常以及售后`。

字段建议：

| 字段 | 说明 |
| --- | --- |
| 售后单号 | 系统生成 |
| 问题类型 | 退货、换货、漏发、设备异常、物流异常、质量问题 |
| 关联平台订单号 | 可选 |
| 客户/收件人 | 可选 |
| 关联 SKU | 可选 |
| 问题描述 | 原因和现象 |
| 退回快递公司/单号 | 可选 |
| 处理方案 | 维修、换新、补发、退款、关闭 |
| 状态 | 待处理、处理中、待寄回、已补发、已解决、已关闭 |
| 责任归类 | 产品、物流、仓库、客户、未知 |
| 处理人 | 当前负责人 |
| 备注/跟进记录 | 时间线 |

需求：

- 新建售后工单。
- 状态流转和跟进记录。
- 可从售后单发起补发出库。
- 可从售后单发起退回入库。
- 支持按状态、问题类型、订单号、快递单号查询。

## 5. 数据模型建议

MVP 建议采用统一库存流水模型，避免直接维护一个容易失真的库存数字。

| 表 | 说明 |
| --- | --- |
| `users` | 管理者账号 |
| `roles` | 角色 |
| `products` | SKU/产品/物料/耗材档案 |
| `warehouses` | 仓库 |
| `locations` | 库位 |
| `stock_balances` | SKU + 仓库 + 库位当前余额，可由流水汇总或缓存 |
| `stock_movements` | 库存流水，所有入库/出库/调整都写入 |
| `inbound_orders` | 入库主表 |
| `inbound_items` | 入库明细 |
| `outbound_orders` | 出库主表 |
| `outbound_items` | 出库明细 |
| `consumables` | 耗材档案，可与 `products` 合并，用类型区分 |
| `bom_rules` | 产品与耗材消耗规则 |
| `after_sales` | 售后工单 |
| `stocktakes` | 盘点主表 |
| `stocktake_items` | 盘点明细 |
| `audit_logs` | 操作日志 |

库存流水方向：

| 类型 | 方向 | 示例 |
| --- | --- | --- |
| 入库 | 增加 | 采购到货、售后退回 |
| 出库 | 减少 | 销售发货、展会、拆样、售后补发 |
| 耗材消耗 | 减少 | 成品生产/到货/出库触发耗材扣减 |
| 盘点调整 | 增加或减少 | 实盘差异审核后调整 |

## 6. Excel 导入策略

首版建议支持一次性初始化导入，而不是持续双向同步 Excel。

导入优先级：

1. `SKU字典` 导入 `products`。
2. `库存` 导入初始库存余额，并生成一批“期初库存”流水。
3. `出库表` 导入历史出库单和出库明细。
4. `KeyPal 2 到货记录` 导入历史入库单。
5. `KeyPal 2 耗材统计记录` 和 `耗材到货记录` 导入耗材档案、耗材到货、耗材消耗规则。
6. `异常以及售后` 尽量结构化导入售后工单，无法自动拆解的内容保留在描述字段。

注意事项：

- `出库表` 中存在文本格式商品记录，历史导入时需兼容 `KeyPal 2 黑色-喷油 1` 和 `SKU*数量+SKU*数量` 两类写法。
- Excel 中部分列是公式结果，系统导入后应改为数据库计算或后端汇总，避免依赖原公式。
- 售后表当前较非结构化，导入时只做基础字段提取，后续在后台补全。

第一阶段先做“能替代 Excel 日常录入”的后台：

2. SKU 管理。
3. 入库登记。
4. 出库登记。

8. 耗材档案、耗材到货、耗材消耗记录。

## 9. 关键业务规则

- SKU 编码全局唯一。
- 所有库存变化必须来自库存流水，不允许直接改库存余额；管理员调整也必须生成调整流水。
- 出库单保存时校验库存是否充足。
- 低库存判断：当前库存 < 安全库存。
- 入库质检合格数量可作为可用库存，不良数量进入异常或不可用库存。
- 售后补发必须生成出库单。
- 售后退回可生成入库单，并标记库存状态。
- 盘点差异必须审核后才影响库存。
- 耗材消耗可以手动登记，也可以由 BOM 规则自动生成。

## 10. 非功能需求

- 操作日志：关键数据修改可追溯。
- 数据导出：SKU、出库、入库、库存、销售汇总、售后支持导出 Excel。
- 权限控制：不同角色看到和操作的菜单不同。
- 数据校验：日期、数量、金额、SKU、订单号等字段格式校验。
- 响应速度：列表页支持分页、搜索和筛选。
- 备份：数据库定期备份。
- 审计：库存调整、盘点审核、删除/停用操作必须留痕。

## 11. 验收标准

- 管理员可以完成注册、登录、退出。
- 可以维护 SKU，并从 Excel 初始化导入 SKU。
- 可以创建一张含多个 SKU 的入库单，库存自动增加。
- 可以创建一张含多个 SKU 的出库单，库存自动减少。
- 库存不足时系统给出明确提示。
- 库存总览能看到当前库存、安全库存、低库存状态。
- 销售汇总能按月、SKU、平台统计出库数量和金额。
- 耗材能登记到货和消耗，并展示当前库存。
- 售后能创建工单、更新状态、记录处理结果。
- 每次入库、出库、调整、盘点审核都能查到操作人和时间。

## 12. 待确认问题

- 是否允许库存为负数，还是必须严格禁止？
- 是否需要多仓库/多库位一开始就启用？
- 耗材消耗是按“成品到货”扣，还是按“成品出库”扣，或两者都支持？
- 销售金额是否全部来自手工填写订单金额，还是需要按 SKU 单价自动计算？
- 售后补发是否计入销售出库统计？
- 是否需要采购订单模块，还是先只做“到货入库记录”？
- 注册账号是否开放给所有管理者自助注册，还是必须由超级管理员邀请？

## 13. 建议开发顺序

1. 搭建项目、数据库、登录注册和权限基础。
2. 建 SKU、仓库、库位、库存流水模型。
3. 实现 SKU 管理和 Excel SKU 导入。
4. 实现入库单、出库单和库存自动计算。
5. 实现库存总览、安全库存预警、库存流水。
6. 实现销售汇总报表。
7. 实现耗材档案、耗材到货、耗材消耗规则。
8. 实现售后工单。
9. 实现盘点和库存调整审核。
